## 🔌 KIẾN TRÚC PLUGGABLE BACKEND (ĐIỂM CỐT LÕI)

**Nguyên tắc**: Frontend KHÔNG được biết backend là gì. Chuyển backend chỉ cần đổi 1 biến môi trường.

### Lớp trừu tượng (Repository Pattern + Adapter)

```
frontend/src/lib/api/
├── contracts/                   # ⭐ INTERFACE chung - SSOT
│   ├── IStudentRepository.ts
│   ├── ITeacherRepository.ts
│   ├── IGradeRepository.ts
│   ├── ICourseRepository.ts
│   └── ... (1 interface / bảng nghiệp vụ)
├── types/                       # Type định nghĩa entity (từ Zod schema)
│   ├── entities.ts              # SinhVien, CanBo, Mon, BangDiem...
│   └── dto.ts                   # Request/Response DTO
├── adapters/                    # ⭐ Implementation có thể HOÁN ĐỔI
│   ├── rest/                    # REST adapter (mặc định)
│   │   ├── RestStudentRepository.ts
│   │   └── ...
│   ├── mock/                    # In-memory mock (dev offline)
│   │   ├── MockStudentRepository.ts
│   │   └── mockDatabase.ts      # faker sinh data
│   ├── graphql/                 # GraphQL adapter (future)
│   └── supabase/                # Supabase adapter (future)
├── factory.ts                   # ⭐ chọn adapter theo env
└── index.ts                     # export useStudentApi(), useGradeApi()...
```

### File `factory.ts` — heart of pluggability
```ts
// frontend/src/lib/api/factory.ts
import type { IStudentRepository } from './contracts/IStudentRepository';
import { RestStudentRepository } from './adapters/rest/RestStudentRepository';
import { MockStudentRepository } from './adapters/mock/MockStudentRepository';

const BACKEND = import.meta.env.VITE_BACKEND_MODE; // 'rest' | 'mock' | 'graphql' | 'supabase'

export const studentRepo: IStudentRepository = (() => {
  switch (BACKEND) {
    case 'mock': return new MockStudentRepository();
    case 'rest': return new RestStudentRepository();
    // case 'graphql': return new GraphQLStudentRepository();
    // case 'supabase': return new SupabaseStudentRepository();
    default: return new RestStudentRepository();
  }
})();
```

### Đổi backend sau này chỉ cần:
1. Tạo thêm adapter mới implement `IStudentRepository` (VD: `FirebaseStudentRepository`)
2. Đăng ký trong `factory.ts`
3. Đổi `VITE_BACKEND_MODE=firebase` trong `.env`
4. **FE component/page KHÔNG sửa 1 dòng nào**

### Quy tắc vàng
- ❌ Cấm `axios.get()` trực tiếp trong component/page
- ❌ Cấm `fetch()` trực tiếp
- ✅ Luôn qua repository: `studentRepo.findById(msv)`
- ✅ TanStack Query wrap repository: `useQuery(['student', id], () => studentRepo.findById(id))`
- ✅ Mọi DTO định nghĩa bằng **Zod schema** → infer TypeScript type → dùng được cả FE & BE

### Shared contract package (tuỳ chọn nhưng nên có)
```
packages/
└── shared/                      # workspace package
    ├── schemas/                 # Zod schemas dùng chung FE + BE
    │   ├── sinhVien.schema.ts
    │   └── ...
    └── types/                   # inferred types
```
→ BE validate request bằng chính schema này, FE validate form cũng schema này → **1 nguồn sự thật**.

### Backend
- **Unit test** (Jest): service, util, validator
- **Integration test** (Supertest): mọi API endpoint
- **E2E** (Playwright): flow login → thao tác → logout

### Frontend
- **Component test** (Vitest + Testing Library): form, table, dialog
- **Adapter test**: đảm bảo mock và rest adapter **cùng pass 1 bộ test** (contract test)
- **E2E** (Playwright): 3 role flows

### Contract test (quan trọng cho pluggable)
```ts
// tests/adapters/contract.test.ts
describe.each([
  ['mock', new MockStudentRepository()],
  ['rest', new RestStudentRepository()],
])('IStudentRepository contract: %s', (name, repo) => {
  it('findById trả về SinhVien đúng shape', async () => {
    const sv = await repo.findById('SV001');
    expect(SinhVienSchema.safeParse(sv).success).toBe(true);
  });
  // ... tests khác cùng áp cho mọi adapter
});
```
→ Đảm bảo mọi backend tương lai đều tuân thủ contract.
